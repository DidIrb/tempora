# Add a Template

Anyone can contribute a template to Tempora. Here is how.

<Steps>

### Fork the repository

Fork [tempora on GitHub](https://github.com/your-org/tempora) and clone it locally.

### Create your template folder

Templates live under `templates/<language>/<template-name>/`:

```
templates/
  typescript/
    your-template/
      tempora.json
      README.md
      ... starter files
```

### Add a tempora.json

Every template needs a `tempora.json` at its root:

```json
{
  "id": "your-template",
  "name": "Your Template Name",
  "language": "typescript",
  "category": "fullstack",
  "description": "A short description of what this template gives you.",
  "tags": ["tag1", "tag2"],
  "version": "1.0.0",
  "nextSteps": [
    "pnpm install",
    "pnpm dev"
  ]
}
```

| Field | Required | Description |
| --- | --- | --- |
| `id` | ✔ | Unique kebab-case identifier |
| `name` | ✔ | Human readable name |
| `language` | ✔ | Primary language e.g. `typescript`, `python` |
| `category` | ✔ | e.g. `fullstack`, `frontend`, `backend` |
| `description` | ✔ | One sentence description |
| `tags` | ✔ | Array of relevant tags |
| `version` | ✔ | Semver string |
| `nextSteps` | — | Commands printed after scaffold. Defaults to `pnpm install` and `pnpm dev` if omitted. |

### Add a README.md

The `README.md` is synced automatically to this docs site. Include the stack, versions, and how to run the project. That is all users need — keep it focused.

### Rebuild and verify

```bash
npm run registry:build
npm run build
tempora init your-template test-output
```

Confirm the scaffold works end to end locally before opening a PR.

### Open a pull request

```bash
git checkout -b feat/add-your-template
git add .
git commit -m "feat(templates): add your-template"
git push
```

<Callout type="info">
  The registry is rebuilt as part of the build process. You do not need to edit registry.json manually.
</Callout>

</Steps>
